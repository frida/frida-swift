## Project Overview

Swift bindings for [Frida](https://frida.re/) dynamic
instrumentation toolkit. Provides Swift-idiomatic async/await
APIs over Frida's C-based FridaCore library. Licensed under
wxWindows Library Licence v3.1.

## Build Commands

```bash
# Build (uses Meson via Python wrapper)
make

# Build with custom prefix
./configure --prefix=/path/to/install && make

# Install (produces Frida.framework on Apple platforms)
make install

# Cross-compile examples
./configure --host=ios-arm64
./configure --host=android-arm64
```

The Makefile delegates to `releng/meson_make.py`. There is no
test suite in this repository.

Swift Package Manager is also supported via `Package.swift`
(pulls FridaCore as a binary XCFramework from GitHub releases).

## Architecture

Three-layer design bridging FridaCore's C API to Swift:

**Layer 1 — FridaCore C API** (`Frida/Frida_Private/`):
Module map and headers exposing the C API. The module map is
generated by `Frida_Private/generate-modulemap.py` from
FridaCore headers.

**Layer 2 — Interop** (`Frida/`):
- `Runtime.swift` — Frida init, GLib main context scheduling,
  dedicated runtime thread
- `AsyncOperation.swift` — Bridges callback-based C API to
  `withCheckedThrowingContinuation`; integrates GCancellable
  with Task cancellation
- `Marshal.swift` — Converts between GVariant/C types and
  Swift types (icons, crash info, parameters, etc.)
- `SignalConnection.swift` — GLib signal connect/disconnect
  helpers

**Layer 3 — Public Swift API** (`Frida/`): Async/await classes
wrapping GObject handles:
- `DeviceManager` → device discovery, `snapshots()` AsyncStream
- `Device` → attach/enumerate/spawn/kill processes, event
  streams
- `Session` → instrumentation session, script creation
- `Script` → JS payload injection, RPC exports, message events
- `Bus` / `Compiler` / `PortalService` / `EndpointParameters`

**Supporting patterns:**
- `AsyncEventSource` — Actor providing shared
  `AsyncStream<Event>` subscriptions with terminal event replay
- `DeviceListModel` — SwiftUI `ObservableObject` wrapper for
  reactive UI
- `GLib/` and `JSONGLib/` — Swift wrappers for GLib and
  JSON-GLib types (DateTime, File, IOStream, etc.)

## Key Conventions

- All Frida operations run on a dedicated GLib main context
  thread; `Runtime.scheduleOnFridaThread` dispatches work there
- Public API classes are `@unchecked Sendable` (they wrap C
  pointer handles and manage thread safety via the GLib main
  context)
- Signal connections are set up in `init` and torn down in
  `deinit` for proper lifecycle management
- Errors are mapped through `Marshal.throwIfError` converting
  GError to the Swift `Error` enum (14 cases)
- Platform support: macOS 11.0+, iOS 13.0+; min Swift tools
  version 5.9

## Dependencies

- **FridaCore**: Binary dependency — XCFramework from
  frida-core GitHub releases (version in `Package.swift`)
- **Build deps**: Defined in `deps.toml` with exact version
  pins (GLib, libffi, pcre2, zlib, etc.)
- **Build infra**: `releng/` git submodule contains Meson
  integration, machine specs, and dependency management
- **Subprojects**: `frida-core` and `frida-gum` as Meson
  subprojects
